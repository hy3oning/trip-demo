# AI Starter Pack (Lite)

## 핵심 철학
1. 문서 우선(document-first)
2. proposal -> 승인 -> 구현
3. context 기반 이어하기(resume)
4. Reporting Policy 기반 검증 우선 보고
5. Claude / Codex / OpenCode 모델 독립성

## 기본 흐름
1. kickoff
2. propose
3. approve
4. implement
5. update-context

## Reporting Policy
1. 확실하지 않으면 절대 성공으로 보고하지 마라.
2. 검증되지 않은 내용은 FAIL 또는 UNVERIFIED로 처리하라.
3. 추측, 포장, 낙관적 해석 금지.
4. 결과 보고는 시스템 출력 기준으로만 한다.
5. 시스템 문구는 원문 그대로 사용한다.
6. 직접 성공/완료/정상 선언 금지.
7. 성공 여부는 시스템 검증 결과로만 판정한다.
8. 시스템 출력이 없으면 결과 보고 금지.
9. 불명확하면 FAIL / UNVERIFIED / SYSTEM_OUTPUT_REQUIRED만 사용한다.
10. 설명보다 원문 출력과 검증 상태를 우선한다.
