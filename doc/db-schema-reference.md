# DB 스키마 레퍼런스

프론트 UI 작업 시 참고할 팀 공유용 DB 명세 초안이다.

- 목적: 나중에 실제 API/DB 기준으로 UI를 확장할 때 컬럼/도메인 누락으로 꼬이지 않도록 기준 보관
- 상태: `REFERENCE`
- 주의: 현재 프로젝트의 실제 백엔드 엔티티/JPA 구조와 1:1로 일치하는 확정 스키마는 아니다
- 주의: 특히 현재 백엔드는 `users`, `lodgings`, `bookings`, `inquiries` 중심 MVP 구조이고, 아래 명세는 그보다 확장된 설계다

## 원본 메모
- 회원/호스트/숙소/객실/쿠폰/예약/결제/리뷰/찜/마일리지/문의/공지까지 포함한 확장형 Oracle DDL
- 프론트에서는 당장 전부 구현 대상으로 보지 말고, 화면 설계 시 필요한 도메인 후보와 컬럼명을 참고하는 용도로 사용
